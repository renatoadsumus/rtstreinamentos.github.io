Desenvolvimento Orientado a Teste - TDD




Objetivo

Apresentar e praticar os principais conceitos das práticas do Extreme Programming (XP). Durante o treinamento iremos exercitar técnicas do desenvolvimento orientado a teste como teste unitário, mocks e integração contínua.

Formato

O treinamento é realizado utilizando a técnica de Dojo e conceitos sobre o tema. Os exemplos abordados são baseados em casos reais e todo o conteúdo produzido durante aula estará disponível no repositório do GitHub. Implementação dos testes automatizados será feita em JAVA ou .NET.

Público Alvo

Pessoas que atuam como desenvolvedores ou possuem um conhecimento básico de programação.

Carga Horária: 16 horas

Conteúdo Programático:

  • TDD
    • Dojo 1: Criando meus primeiros testes automatizados
    • O que é TDD?
    • Como é o fluxo de trabalho com TDD?
    • Benefícios em usar Baby steps
  • Mock
    • Dojo 2: Criando testes sem usar Mock
    • Dojo 3: Refactoring Dojo 2 usando Mock
    • O que é Mock?
    • Quando usar Mocks?
    • Mocks em JAVA e .NET
  • Teste Integrado
    • Dojo 4: Criando testes integrados com banco de dados
    • O que é um teste integrado?
  • Jenkins
    • O que é o Jenkins
    • Como Instalar
    • Criar o Primeiro Projeto no Jenkins acessando SVN e Github
    • Configurar Testes Unitários
    • Configurar Envio de Email
    • Agendar Execução

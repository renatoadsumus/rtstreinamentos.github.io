Infraestrutura como código com Docker, Puppet e Vagrant




Objetivo

Ensinar os principais conceitos de Devops e como usar as ferramentas de infraestrutura como código. Ao final do curso você será capaz criar container, criar e provisionar máquina virtual usando as ferramentas: Docker, Puppet e Vagrant.

Formato

O treinamento é realizado utilizando a técnica de Dojo e conceitos sobre o tema. Os exemplos abordados são baseados em casos reais e todo o conteúdo produzido durante aula estará disponível no repositório do GitHub.

Público Alvo

Analistas de infraestrutura, pessoas relacionadas a área de teste de qualidade de software, desenvolvedores ou pessoas interessadas em aprender ferramentas de automatização.

Carga Horária: 16 horas

Conteúdo Programático:

  • Introdução Devops
  • Diferença entre Integração Contínua, Entrega Contínua e Implantação Contínua
  • Entendendo a diferença entre Virtualização x Container
  • O que é infraestrutura como código
  • Benefícios da infraestrutura como código
  • Conhecendo e Vagrant e suas principais configurações
  • Entendendo o que é provisionamento
  • Usando Scripts e Puppet para provisionamento
  • O que é Docker?
  • Conhecendo repositórios de imagem
  • Criando meu primeiro container
  • Conhecendo as principais instruções do Docker
  • Aprendendo a instalar o Jenkins
  • Usando o jenkins para realizar deploy de um container
  • Exercício práticos com necessidades comuns do dia-a-dia
